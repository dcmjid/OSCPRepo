HTTP Tunneling is a technique where a payload protocol is encapsulated within the HTTP protocol (usually in a GET or POST request). When behind a HTTP proxy, you can use the CONNECT HTTP method to ask the proxy server to forward a connection to a specific destination. The proxy then makes the connection on behalf of the client:

root@kali:~# nc -vvn 192.168.1.100 8888
(UNKNOWN) [192.168.1.100] 8888 (?) open
CONNECT 10.10.10.200:80 HTTP/1.0

HTTP/1.0 200 Connection established
Proxy-agent: tinyproxy/1.8.3

HEAD / HTTP/1.0
