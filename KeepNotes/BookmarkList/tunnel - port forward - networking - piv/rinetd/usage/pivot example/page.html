Internal nework 1.2.3.4 only allows outbount TCP 53 and want to connect to a web server on 80.
Setup the following in rinetd.conf on the attacker machine:

# bindaddress bindport connectaddress connectport
<attacker ip> 53 <web server> 80

The internal network can now issue http://<attacker ip> and be forwarded to the web server on port 80
